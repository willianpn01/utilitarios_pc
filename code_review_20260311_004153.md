## 📄 `app\core\app_paths.py`

### Problema 1 — Linhas 23–41 e 44–53: `os.makedirs` em cada chamada de leitura

- ⚠️ **Descrição:** `get_data_dir()` chama `os.makedirs()` toda vez que é invocada, mesmo que o diretório já exista. `get_undo_history_dir()` chama `get_data_dir()` internamente, gerando chamadas em cascata ao sistema de arquivos (syscalls desnecessárias). Funções como `get_watcher_config_path()` e `get_clipboard_db_path()` também chamam `get_data_dir()` repetidamente.

- ✅ **Sugestão:** Usar `functools.lru_cache` para memoizar o resultado de `get_data_dir()` (o diretório raramente muda) e separar a criação do diretório da obtenção do caminho:

```python
import functools

@functools.lru_cache(maxsize=1)
def get_data_dir() -> str:
    custom = os.environ.get('UTILITARIOS_DATA_DIR')
    if custom and os.path.isabs(custom):
        os.makedirs(custom, exist_ok=True)
        return custom
    home = os.path.expanduser("~")
    data_dir = os.path.join(home, ".utilitarios")
    os.makedirs(data_dir, exist_ok=True)
    return data_dir
```

Isso garante que a syscall de `makedirs` e `expanduser` ocorram apenas uma vez por execução.

### Problema 2 — Linhas 67–72: `get_all_data_paths()` causa múltiplas chamadas encadeadas

- ⚠️ **Descrição:** `get_all_data_paths()` chama 4 funções diferentes, cada uma delas chamando `get_data_dir()` individualmente. Mesmo com `lru_cache`, o encadeamento cria overhead de chamada de função desnecessário.

```python
def get_all_data_paths() -> dict:
```

- ⚠️ **Descrição:** Cada chamada a `load_setting()` abre o arquivo JSON, lê-o e faz parse completo. Cada chamada a `save_setting()` faz isso e ainda reescreve o arquivo inteiro.         return _settings_cache

```python
        _settings_cache = {}
        return _settings_cache

def save_setting(key: str, value: Any) -> None:
    global _settings_cache
    settings = _load_all()
    settings[key] = value
    p = _settings_path()
    try:
        with open(p, 'w', encoding='utf-8') as f:
```
